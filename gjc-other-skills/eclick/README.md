# gjc-eclick 사용법

`gjc-eclick`은 월간 eClick 실적을 매일 누적해서 관리하고, 월말에 복사/붙여넣기용 Excel을 생성하는 글로벌 GJC 스킬입니다.

## 설치 위치

```text
~/.gjc/agent/skills/eclick/SKILL.md
~/.gjc/agent/skills/eclick/README.md
```

## 저장 위치

월별 데이터는 전역 캐시에 저장됩니다.

```text
~/.gjc/eclick/YYYY-MM/
  schedule.json
  entries.jsonl
  sources.jsonl
  rules.json
  eclick_YYYY_MM.xlsx
```

이 파일들이 이전 기록 역할을 하므로, 새 세션에서도 이어서 작업할 수 있습니다.

## 기본 사용 흐름

### 1. 월초 계획 생성

```text
/skill:eclick 2026년 7월 시작.
비중 AX 0.75, 케미칼 0.25.
연차 7/14, 7/15.
오전반차 7/23.
공휴일 없음.
매주 금요일 데이터파트 주간회의 1h.
```

스킬이 자동으로:
- 1일부터 말일까지 기준표 생성
- 주말 0h 처리
- 연차/반차/연휴 반영
- AX/케미칼 목표 시간 계산
- 빈 엑셀 생성

### 2. 매일 작업 추가

```text
/skill:eclick 7/8 오늘 한 일:
토목현장 HOME 관련 문서 확인했고, 팔란티어 구조를 파악했어.
https://example.com/doc
```

스킬이 자동으로:
- 링크/문서 분석
- 업무 제목 추천
- 처리내용 작성
- 1~3개 티켓으로 분리
- 시간 배분
- 누적 현황 검증
- Excel 재생성

### 3. 현황 확인

```text
/skill:eclick 현황
```

예시 출력:

```text
2026-07 eClick 현황

AX:       72 / 138h
케미칼:   18 / 46h
전체:     90 / 184h

남은 근무일: 11일
남은 시간: 94h
필요 AX: 66h
필요 케미칼: 28h
상태: 정상
```

### 4. 리밸런싱

```text
/skill:eclick 7월 리밸런싱해줘
```

기존 업무 사실은 유지하면서 AX/케미칼 목표, 날짜별 8h, 하루 3티켓 제한에 맞게 시간과 지원 표기를 조정합니다.

### 5. 수정

```text
/skill:eclick 7/8 두 번째 항목 제목을 "토목현장 HOME 업무 분석"으로 바꿔줘
/skill:eclick 마지막 입력 취소
```

수정 이력은 `entries.jsonl`에 남습니다.

### 6. 월말 마감

```text
/skill:eclick 7월 마감
```

검증 항목:
- 전체 시간 일치
- AX 목표 일치
- 케미칼 목표 일치
- 날짜별 시간 일치
- 하루 최대 3티켓 준수
- 처리내용 누락 없음

## 중요 규칙

### AX/케미칼

앞으로 대부분 실제 업무는 AX입니다. 케미칼은 실제 별도 업무가 아니라 0.25 M/M 배분용일 수 있습니다.

예시:

| 실제 업무 | AX 표기 | 케미칼 표기 |
|---|---|---|
| 토목현장 HOME 팔란티어 업무 | 토목현장 HOME 팔란티어 업무 | 토목현장 HOME 팔란티어 업무 지원 |
| 교육자료 제작 | 교육자료 제작 참여 | 교육자료 제작 지원 |

### 하루 티켓 제한

하루 최대 3개 티켓입니다.

### 제목/처리내용

- 제목은 짧고 대표성 있게.
- 세부 내용은 Excel 아래쪽 `처리내용 상세`에 작성.
- 제목 클릭 시 처리내용으로 이동합니다.

## Excel 구조

한 시트 안에 모두 들어갑니다.

```text
[상단] 실적 리스트
No | 날짜 | 요일 | 제목 | M/H | 구분 | 목표(h) | 할당(h) | 비고

[하단] 처리내용 상세
- 제목별 상세 내용
- 상단 제목과 하단 상세가 하이퍼링크로 연결
```

## 링크/문서 분석

작업 설명에 링크, 파일, 이미지, PDF, Excel을 넣으면 스킬이 읽고 요약합니다.

```text
/skill:eclick 7/10 작업 추가.
아래 문서 보고 HOME 데이터 구조 파악했어.
https://...
```

처리내용은 업무용 문장으로 정리됩니다.

## 추천 사용 패턴

매일 자세히 쓰지 않아도 됩니다.

```text
/skill:eclick 7/11 오늘 한 일:
HOME 인수인계 받았고, 온톨로지 쪽 문서 좀 봤고, 교육자료 피드백 조금 했어.
```

스킬이 적절히 나눕니다.

## 자체 성장 방식

사용자가 규칙을 고치면 다음에 반영합니다.

예:
- "케미칼은 SNOP가 아니라 AX 지원으로만 써줘"
- "처리내용은 너무 개발자스럽지 않게 비즈니스 문장으로 써줘"
- "하루 티켓은 3개까지만"

이런 규칙은 스킬 문서 또는 월별 `rules.json`에 반영해서 다음 달에 재사용합니다.


## 백업/복구용 설치

이 저장소를 새 PC 또는 새 Windows 계정의 D 드라이브에 받은 뒤 아래 명령을 실행합니다.

```bash
bash /mnt/d/etc/agy-con/gjc-other-skills/eclick/install.sh
```

설치 스크립트가 수행하는 작업:
- `~/.gjc/agent/skills/eclick/` 생성
- `SKILL.md`, `README.md` 복사
- `/eclick` shortcut용 `~/.gjc/agent/commands/eclick.md` 생성

설치 후 GJC를 다시 시작하면 `/eclick`이 slash 목록에 표시됩니다.

## 월별 데이터 백업

스킬 본체와 월별 데이터는 별도입니다. 월별 누적 데이터까지 보존하려면 아래 폴더를 D 드라이브나 Git 저장소 밖 백업 폴더로 복사합니다.

```text
~/.gjc/eclick/
```

복구 시 같은 위치로 되돌립니다.

```bash
mkdir -p ~/.gjc
cp -a /mnt/d/etc/eclick-backup/.gjc/eclick ~/.gjc/eclick
```
