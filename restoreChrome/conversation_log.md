# 크롬 프로필 자동 복원 - 작업 대화 기록

> **날짜**: 2026-06-10  
> **원본 Gemini 대화**: https://gemini.google.com/share/83c2dfecf0d9  
> **구현 도구**: Antigravity (Claude Opus 4.6)

---

## 1. 문제 상황

회사 보안정책으로 인해 매일 밤 크롬의 쿠키/로그인 데이터가 삭제됨.  
그 결과 매일 아침 크롬을 열면 **"일시중지됨"** 상태가 되어 모든 계정에 다시 로그인해야 하는 반복 작업 발생.

## 2. Gemini 대화 내용 (원본)

### 사용자 질문 1

> 보안정책상 자꾸 크롬의 프로필을 삭제해서 그런지 일시중지됨으로 변해.
> 이걸 막을순 없을텐데 아침마다 이전 프로필을 다시 복원하는 방법이 있을까?
> 검색도 해보고 해서 정확하게 이야기를 해주면 좋겠어

### Gemini 답변 1

보안 프로그램이 매번 크롬 세션이나 쿠키 데이터를 초기화해버려서 자꾸 일시중지됨 상태로 풀려버리는 현상이구나. 매번 다시 로그인하고 동기화 켜는 거 진짜 번거롭지.

사내 보안 정책 자체를 우리가 우회할 수는 없겠지만, 크롬이 정상적으로 로그인된 상태의 프로필 폴더를 따로 백업해두고 아침마다 크롬을 켜기 전에 복원하는 스크립트를 쓰면 훨씬 편해질 거야.

#### 백업 스크립트 (chrome_backup.bat)

```bat
@echo off
echo 크롬을 종료하고 프로필 백업을 시작할게.
taskkill /F /IM chrome.exe /T >nul 2>&1
timeout /t 2 >nul
xcopy "%LOCALAPPDATA%\Google\Chrome\User Data\Default" "C:\ChromeProfileBackup\Default" /E /C /I /Y /H
echo.
echo 백업이 완료되었어!
pause
```

#### 복구 스크립트 (chrome_restore.bat)

```bat
@echo off
echo 크롬을 종료하고 이전 프로필로 복원을 시작할게.
taskkill /F /IM chrome.exe /T >nul 2>&1
timeout /t 2 >nul
xcopy "C:\ChromeProfileBackup\Default" "%LOCALAPPDATA%\Google\Chrome\User Data\Default" /E /C /I /Y /H
echo.
echo 복원이 완료되었어! 이제 크롬을 실행해봐.
pause
```

#### 적용하는 팁
- 퇴근하기 전에 크롬이 정상 상태일 때 백업 스크립트를 한 번 실행해서 데이터를 C드라이브 안의 백업 폴더로 안전하게 복사해둬.
- 다음날 아침에 출근해서 크롬을 켜기 전에 복구 스크립트를 먼저 실행하고 크롬을 켜면, 어제 퇴근 전 상태 그대로 유지되어 있을 거야.

### 사용자 질문 2

> 그럼 일단 다시 로그인을 할건데 일시중지됨이 되는걸 구현을 하고 (보안상 초기화 되는걸 미리 구현) 다시 그걸 복원하는걸 해보려고 하는데 어떻게 할까?

### Gemini 답변 2

보안 프로그램이 크롬을 '일시중지됨'으로 만드는 원리는 간단해. 크롬의 로그인 유지 정보(쿠키 및 세션 데이터)만 날려버리는 거거든. 이걸 우리가 수동으로 똑같이 재현해 볼 수 있어.

#### 1단계: 정상 상태 백업하기
1. 크롬에 로그인해서 '일시중지됨'이 아닌 정상 동기화 상태를 만들어줘.
2. 크롬을 닫거나 켠 상태에서 `chrome_backup.bat` 파일을 실행해.
3. 백업 완료 메시지가 뜰 때까지 기다려. (이제 정상 상태가 C드라이브에 저장됐어)

#### 2단계: '일시중지됨' 상태 강제로 만들기 (보안 정책 시뮬레이션)
1. 크롬을 열고 주소창에 `chrome://settings/clearBrowserData` 를 입력하고 엔터를 쳐.
2. [인터넷 사용 기록 삭제] 창이 뜨면 기간을 '전체 기간'으로 맞춰.
3. 다른 체크박스는 다 해제하고, '쿠키 및 기타 사이트 데이터'에만 체크해.
4. [데이터 삭제] 버튼을 눌러.
5. 우측 상단의 크롬 프로필 아이콘을 눌러봐. '일시중지됨'으로 바뀌어 있을 거야! (시뮬레이션 성공)

#### 3단계: 스크립트로 복원 테스트하기
1. 켜져 있는 크롬 창을 모두 닫아줘. (크롬이 켜져 있으면 파일이 덮어씌워지지 않아)
2. 바탕화면에 만들어둔 `chrome_restore.bat` 파일을 실행해.
3. 복원 완료 메시지가 뜨면, 다시 크롬을 켜봐.

#### 4단계: 결과 확인
오른쪽 위 프로필 아이콘을 봤을 때 '일시중지됨'이 사라지고 다시 정상적으로 동기화가 연결되어 있다면 테스트 대성공이야!

## 3. 실제 구현 (Antigravity)

Gemini 대화를 바탕으로 실제 구현을 진행. 원본 대비 개선 사항:

### 개선점

1. **전체 프로필 복사**: 여러 Google 계정이 로그인된 상태를 온전히 보존하기 위해 Default 폴더 전체 + `Local State`를 통째로 xcopy
2. **UTF-8 BOM 인코딩**: cmd에서 한글이 깨지지 않도록 파일 인코딩 처리
3. **작업 스케줄러용 무인 스크립트**: `pause`/`set /p` 없는 silent 버전 별도 생성
4. **로그 기록**: 스케줄러 실행 시 `backup_log.txt` / `restore_log.txt`에 타임스탬프 기록
5. **시뮬레이션 테스트 자동화**: 쿠키/로그인 데이터만 선별 삭제하는 테스트 스크립트

### 환경 확인

```
크롬 프로필 경로: %LOCALAPPDATA%\Google\Chrome\User Data\Default
프로필 크기: ~391 MB
프로필 파일 수: 2,835개
추가 프로필: 없음 (Default만 사용, 여러 Google 계정은 이 안에 포함)
```

### 4단계 테스트 실행 결과

| 단계 | 작업 | 결과 |
|------|------|------|
| 1단계 | 정상 상태 백업 | ✅ 2835파일, 391MB 백업 완료 |
| 2단계 | 보안 초기화 시뮬레이션 (쿠키/로그인 파일 10개 삭제) | ✅ 삭제 완료 |
| 3단계 | 스크립트로 복원 | ✅ 2835파일 복원, 인증 파일 4종 확인 |
| 4단계 | 크롬 실행하여 정상 확인 | ✅ 사용자 직접 확인 - "굿~ 복원됐다!!" |

### 작업 스케줄러 등록

```
ChromeProfileBackup:  평일 오후 5:30 → chrome_backup_silent.bat
ChromeProfileRestore: 평일 오전 8:15 → chrome_restore_silent.bat
```

## 4. 최종 파일 구성

```
restoreChrome/
├── chrome_backup.bat          ← 수동 백업 (대화형)
├── chrome_restore.bat         ← 수동 복원 (대화형)
├── chrome_backup_silent.bat   ← 스케줄러용 백업 (무인, 로그 기록)
├── chrome_restore_silent.bat  ← 스케줄러용 복원 (무인, 로그 기록)
├── test_simulate_reset.bat    ← 보안 초기화 시뮬레이션 테스트
├── README.md                  ← 설치/사용법 가이드
└── .gitignore                 ← backup/ 및 로그 제외
```

## 5. 다른 PC에서 사용하기

1. 이 레포를 클론하거나 `restoreChrome` 폴더를 다운로드
2. 원하는 위치에 배치 (예: `C:\Users\{사용자}\Downloads\restoreChrome\`)
3. 크롬 정상 로그인 상태에서 `chrome_backup.bat` 실행
4. `test_simulate_reset.bat`으로 시뮬레이션 테스트
5. `chrome_restore.bat`으로 복원 확인
6. 작업 스케줄러 등록 (README.md 참고)
